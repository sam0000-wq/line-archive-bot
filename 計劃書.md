# LINE 群組訊息歸檔系統 — 計劃書

> 版本：v1.0　　最後更新：2025-07

---

## 一、專案目標

將特定 LINE 群組中收到的新訊息，透過 Python 腳本自動歸檔至 `.xlsx` 檔案，並每天定時寄送至指定信箱。

---

## 二、系統架構

```
LINE 群組成員發訊息
        ↓
   LINE Platform
        ↓  (HTTPS Webhook POST)
  Flask Webhook Server
  （部署於 Render）
        ↓
  簽名驗證（Channel Secret）
        ↓
  訊息分類邏輯
  ├── critical: 開頭 → critical sheet
  ├── warning:  開頭 → warning sheet
  └── 其餘訊息         → others sheet
        ↓
  openpyxl 寫入 .xlsx
        ↓
  每日 20:00（台灣時間）
  Gmail SMTP 寄送報告
```

## 六、各檔案職責說明

### `app.py` — Flask Webhook Server

- 監聽 LINE Platform 的 Webhook POST 請求
- 驗證 `X-Line-Signature`（使用 Channel Secret）
- 篩選目標群組（`TARGET_GROUP_ID`）
- 呼叫 `archiver.py` 進行分類歸檔
- APScheduler 每日 `20:00 Asia/Taipei` 自動呼叫 `mailer.py`
- 提供 `POST /send-report` 手動觸發端點
- 首次收到群組訊息時，自動 log `source.groupId`

### `archiver.py` — 歸檔模組

- 接收訊息事件（timestamp、sender_name、sender_user_id、message）
- 依前綴分類寫入對應 sheet：
  - `critical:` → `critical` sheet
  - `warning:` → `warning` sheet
  - 其餘 → `others` sheet
- 使用 `openpyxl`，若檔案不存在則自動建立並加上表頭
- 檔名格式：`line_archive_YYYYMMDD.xlsx`

### `mailer.py` — 寄信模組

- 使用 Gmail SMTP（`smtp.gmail.com:587` + TLS）
- 附件寄送當日 xlsx 檔案
- 支援排程呼叫與手動呼叫

### `.env.example` — 環境變數範本

```env
# LINE
LINE_CHANNEL_SECRET=your_channel_secret_here
LINE_CHANNEL_ACCESS_TOKEN=your_access_token_here
TARGET_GROUP_ID=                  # Bot 加入群組後從 log 取得

# Gmail SMTP
GMAIL_USER=itsamliu2025@gmail.com
GMAIL_APP_PASSWORD=your_app_password_here
REPORT_TO_EMAIL=itsamliu2025@gmail.com

# 時區
TZ=Asia/Taipei
```

## 八、Render 部署步驟

### 1. 建立 Render Web Service

1. 前往 [https://render.com](https://render.com)，新增 **Web Service**
2. 連結你的 GitHub repo（或直接上傳）
3. 設定：
   - **Build Command**：`pip install -r run/requirements.txt`
   - **Start Command**：`gunicorn run.app:app`
   - **Environment**：Python 3

### 2. 設定環境變數

在 Render Dashboard → Environment 中加入：

| Key | Value |
|-----|-------|
| `LINE_CHANNEL_SECRET` | `4695ff0de...`（實際值） |
| `LINE_CHANNEL_ACCESS_TOKEN` | `cCchQly0oKnd...`（實際值） |
| `TARGET_GROUP_ID` | 待填（第一步先留空，Bot 加入群組後從 log 取得） |
| `GMAIL_USER` | `itsamliu2025@gmail.com` |
| `GMAIL_APP_PASSWORD` | `uulfdwvlldolvlhw` |
| `REPORT_TO_EMAIL` | `itsamliu2025@gmail.com` |
| `TZ` | `Asia/Taipei` |

### 3. 取得 Webhook URL

部署完成後，Render 會給你一個 URL，格式如：
```
https://your-service-name.onrender.com/callback
```

### 4. 設定 LINE Webhook URL

1. 前往 [LINE Developers Console](https://developers.line.biz/console/channel/2005988474/messaging-api)
2. 將 Webhook URL 更新為：`https://your-service-name.onrender.com/callback`
3. 開啟 **Use webhook**
4. 按下 **Verify** 測試連線

### 5. 取得 Target Group ID

1. 將 Bot（`@759jrnvd`）邀請加入目標群組
2. 在群組中發送任意訊息
3. 查看 Render Log，找到 `[GROUP ID DETECTED]` 的輸出
4. 將該 Group ID 填入 Render 環境變數 `TARGET_GROUP_ID`
5. 重新部署（或 Render 會自動重啟）

## 九、訊息分類邏輯

```python
text = message.lower().strip()

if text.startswith("critical:"):
    sheet = "critical"
elif text.startswith("warning:"):
    sheet = "warning"
else:
    sheet = "others"
```

> 比對不分大小寫（統一轉小寫後比對）

## 十、xlsx 檔案格式

### 欄位定義

| 欄位名稱 | 說明 | 範例 |
|----------|------|------|
| `timestamp` | 訊息時間（UTC+8） | `2025-07-01 14:32:10` |
| `sender_name` | 發送者顯示名稱 | `小明` |
| `sender_user_id` | 發送者 LINE User ID | `Uxxxxxxxxxxxxxxx` |
| `message` | 訊息原文 | `critical: 系統異常` |

### Sheet 說明

| Sheet 名稱 | 收錄條件 |
|------------|----------|
| `critical` | 訊息以 `critical:` 開頭（不分大小寫） |
| `warning` | 訊息以 `warning:` 開頭（不分大小寫） |
| `others` | 其餘所有訊息 |
